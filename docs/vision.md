# Product Vision

## 一句话定位

这是一个在 Codex 中长期协作的加密交易搭子：它不是替用户拍板的信号机，也不是被动记账的日志工具，而是一个和用户共享上下文、共享判断、共享复盘的投资协作系统。

## 为什么值得存在

真实交易里最稀缺的，往往不是观点本身，而是连续、可回看、可修正的判断过程。

今天的大多数工具只能覆盖其中一小段：

- 要么擅长给一个当下看起来很聪明的回答
- 要么擅长保存零散记录
- 要么擅长展示市场数据

但用户真正缺的是另一种东西：

- 当问题已经指向真实资金决策时，系统能先补齐必要上下文，再给出足够严谨的判断
- 一次判断不是说完就结束，而是能自然进入跟踪、修正、结束与复盘
- 后续回看时，用户能知道自己当时为什么那样想、系统为什么那样判断、最后错在判断、执行还是运气

这个产品值得存在，是因为它想补上的不是一个答案，而是一条完整的决策协作链。

## 为什么现在

这件事现在值得做，有三个原因：

- 第一，市场信息已经可以被更稳定地工具化接入。加密行情、宏观变量和跨市场数据越来越容易通过 API 进入同一个工作上下文。
- 第二，像 Codex 这样的环境已经不只是聊天窗口，而是可以调用 skill、读取历史、维护 workspace 的持续工作台，适合承接连续判断而不是一次性问答。
- 第三，加密市场本身高波动、全天候、跨市场联动强，用户既容易被噪音牵着走，也最需要一个能持续校准判断的协作系统。

换句话说，不是“现在刚好能做一个更聪明的问答机器人”，而是现在终于有条件把判断、跟踪、复盘放进同一个可持续的工作环境里。

## 为什么是这个系统

这个产品不是靠“大模型更聪明”成立，而是靠下面这组组合成立：

- 用户可以用很短的入口发起问题
- 系统可以主动调用工具重建当下需要的市场上下文
- 系统可以把当次判断放进同一 workspace 的连续链路里
- 系统可以在链路结束后复盘，并把复盘结果带回后续判断

也就是说，这个系统的独特性不在于它会回答，而在于它能把回答放进持续协作关系里。

## 目标用户

当前面向的不是“愿意用 Codex 的人”，而是下面这类真实用户：

- 在加密货币市场上做真实资金决策的个人投资者或交易者
- 有自己的仓位、偏好和主观判断，但市场信息、图表、新闻、宏观变量和历史想法分散在多个地方
- 会同时参考加密市场内部结构与外部市场背景，但很难在真正出手前把这些信息收束成一致判断
- 经常在“临时有感觉就做”“做完以后说不清为什么”“复盘时只能看盈亏”之间来回摆动

他们在没有这个产品之前，不是没有信息，而是信息太多、链路太断、复盘太弱。

## 协作是什么意思

这里说的“协作”不是一个好听的拟人化词，而是明确指双方都在贡献判断过程。

用户负责提供：

- 自己当前真正关心的问题
- 持仓、风险承受、时间尺度、例外约束
- 最终是否采纳某个计划，以及真实执行结果

系统负责提供：

- 形成判断前所需的上下文补齐
- 结构化判断、条件式计划与风险边界
- 对后续变化的持续跟踪
- 对已结束链路的复盘与归因整理

因此，这个产品不是替代用户做决定，而是让用户和系统共同维护一条持续更新的判断链。用户贡献真实意图与真实动作，系统贡献上下文重建、结构化判断与连续跟踪；两者缺一不可。

## 复盘让谁进化

复盘不是一句口号，它至少服务两个主体的进化。

第一，用户要进化。

- 用户不只看到这次赚没赚钱，而是能更清楚地区分自己当时的前提、触发条件、执行纪律与情绪偏差
- 用户应逐渐形成更稳定的出手标准，而不是每次重新被市场牵着走

第二，系统也要进化。

- 这里说的不是底层模型参数自动训练
- 而是系统在 workspace 内的判断方法、优先级、阈值、关注对象与复盘框架要能被持续校准
- 复盘后的结论如果足够稳定，应影响后续判断方式，而不是停留在“记了一笔对错”

所以，复盘的目标不是证明谁对谁错，而是让用户和系统都更少重复犯同一种错误。

## 核心价值

这个产品当前提供的核心价值有五点：

- 短入口，不简化判断
- 重视上下文重建，而不是依赖一次对话窗口里的临时记忆
- 把“看市场 -> 出建议 -> 后续跟踪 -> 复盘”视为一条连续链路
- 允许多个并行判断共存，而不是强行压成单一路径
- 只在真正需要持续管理时才进入记录与维护，避免把存储本身当成目标

## 北极星指标

当前阶段的北极星指标先定义为：

用户是否愿意在同一 workspace 内持续完成“提问 -> 采纳判断 -> 跟踪更新 -> 复盘 -> 下一轮判断更稳”这条闭环。

这比单次回答是否看起来聪明更重要，因为它更能说明产品是否真的在帮助真实决策能力持续改善。

## 产品边界

当前明确不把产品定义成下面这些东西：

- 不是单次给买卖点的信号机
- 不是默认每次分析都要落库的记录系统
- 不是脱离上下文就直接输出结论的快问快答机器人
- 不是当前阶段就要预先固定完整 skill 结构、记录模型或策略流程的系统
- 不是一个对所有市场都提供同等体系化交易能力的通用投资平台

当前范围需要区分两层：

- 系统会跨市场分析，不只读取加密数据，也会逐步接入 A 股、港股、美股、大宗商品、汇率、美债收益率、恐慌指数、通胀等更广泛的市场信息
- 但当前真正被体系化建设的主战场，仍然是加密领域，尤其是围绕加密货币交易协作的正式判断、跟踪、复盘与方法演化
- 其他市场可以进入分析上下文，也可以影响判断背景；只是现阶段不会像加密交易那样，默认拥有同样完整、同样结构化的交易系统承接

## 当前产品判断

围绕这个 vision，当前已经比较明确的上层判断有：

- 产品重心是判断质量，而不是存储密度
- 在单笔正式判断之前，系统应先形成共享的决策底板，再把结论投影到具体机会
- 不是每次市场讨论都要升级成正式链路，只有被采纳、需要持续跟踪、已经持仓或明确要求复盘时，才进入后续维护
- 复盘不是附属动作，而是用户校准自己、系统校准方法的必要组成
