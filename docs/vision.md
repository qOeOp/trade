# Product Vision

## 产品形态

产品主体是一组运行在 agent 工作区里的 skill，通过 Claude Code、Codex、Gemini CLI 直接调用，不做从头到尾包办的独立 app。

这些 skill 保持单一职责，由 agent 负责调度和编排。当前先把判断记录、上下文召回、research、策略生成、回测、复盘这些核心能力跑通，再考虑更重的界面。

在这层能力之上，可以有单独看板，展示长期积累的数据，并支持基本查看、增删改查。

长期数据更适合放在服务器侧统一存储；本地 agent 负责读写，看板读取同一份数据做展示。这样能力层、数据层和视图层可以协同，但不绑成单体。

电脑和手机共用同一个远程工作区和持久化状态，不分叉。主战场是加密交易，其他市场只作为背景参考或单次分析，不进入完整交易链路。

## 产品主线

产品主线是一条连续的交易链：观察、决策、执行、管理、复盘、回测、迭代。重点是保证链条不中断，用户下次回来还能继续。

这七个阶段是主线，不表示真实交易一定线性推进。实际使用里，用户会在几个状态之间反复往返，先承认这种往返存在，不急着把它写成过细制度。

用户每次进入系统，先基于现有持仓、挂单、既有判断与最新市场数据恢复并对账当前交易链，再进入当前所处阶段继续推进。

至少有两类常见循环：

- 开仓前，观察和决策会反复往返；没有操作建议、不建议操作、撤单、改单或放弃计划后，都回到观察和决策；有挂单但未成交时，进入监控
- 持仓后，管理意味着继续观察、继续决策，并继续执行新的动作，例如加仓、减仓、调杠杆、止损、止盈；如果又挂出新单但未成交，再次进入监控

订单也有自己的生命周期：创建、挂出、等待成交、部分成交、全部成交、撤销、改价、失效。订单状态变化时，主线所处阶段往往也会跟着变化。

复盘不会直接变成长期规则；先回测，回测通过后再进入迭代。

策略也不应只是一份全局单体规则。更合理的形态是一个带分支和版本管理的策略池：不同分支可对应不同交易标的或不同市场环境；主线每次只读取当前语境下适用的生效分支版本，而投研、复盘产出的候选策略先入池，再经回测和迭代决定是否升格。

## 产品支线

除了围绕真实交易记录持续运转的主线，系统还需要两条能独立运行、也能接回主线的功能：投研 和 回放。投研 是整个系统的 research 中枢，回放 负责从历史市场数据中选取起始时间点，提供可供回测使用的市场回放语境。
主线、投研 和 回放 可以独立存在。投研 的输出有时会进入“策略生成 -> 回测 -> 迭代”，有时则直接回接主线，服务当下的观察、决策和管理。

### 投研

投研 不依赖实盘持仓触发，负责消化外部材料、整理不同来源的想法，并把它们转成可供主线使用的研究结论；其中一部分会继续沉淀成可回测的策略。

投研 的输出不只有策略，还可以是：

- 对某个标的当前语境的研究判断
- 对宏观、流动性、地缘政治、监管或市场报告的解读
- 对其他市场变化如何影响加密标的的解释
- 候选假设、观察重点或风险提示

策略来源包括：

- 用户自己提出的假设
- agent 发散出的假设
- 社区流传的策略
- 网络、论坛、论文里的公开策略

### 回放

回放 不直接回测策略，而是负责从历史市场数据中选取时间点开始，生成可供回测使用的市场回放语境。

回测使用的策略来源：

- 主线复盘
- 投研

回测输入：

- 用户自己的历史交易记录
- 从历史市场数据中选取时间点开始的市场回放
