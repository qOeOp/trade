# Product Vision

## 一句话定位

把我临时、断裂、不可回看的交易判断，变成可延续、可回看、可修正的判断链的加密交易工作台。

当前不是面向泛用户的 SaaS，而是运行在 agent 工作区里的高级 skill 集合，给开发者自己长期使用。

## 真正缺什么

我不缺信息，也不缺观点。缺的是：真实资金决策发生时，没有一条连续、可回看、可修正的判断链。

断裂最常见的三个时刻：

- 手上有仓位，但已经记不清当初为什么那样判断
- 交易结束后说不清是判断错了、执行乱了，还是运气差
- 想跟进一个标的，却要从头重建上次的思路

今天这些内容散落在 TradingView + 笔记 + 群消息之间，信息很多，判断过程是断的。

## 为什么是现在

不是因为模型更会说话，而是第一次具备了把这条链真正接起来的条件：agent 能主动调用工具拿市场数据，工作区能承接历史上下文而不只是一次性对话，加密市场全天候高波动，最需要连续判断而不是临时灵感。

## 它最小长什么样

用户打开时，第一眼看到的不是"今天想问什么"，而是当前有哪些活跃判断链、每条链现在看到哪一步、最近发生了什么变化、下一次为什么值得回来。

比如丢一句"ETH 现在还拿吗"，系统不该像第一次见到我一样从头回答，而应该先检索相关旧链，判断这是旧链延续还是新链开始；旧链则先把上次判断、之后变化和当前缺口接起来，再只问最少一个真正影响结论的问题；新链则立刻形成一条之后还能继续更新和复盘的链，而不是让这次判断再次散掉。

**冷启动阶段**（没有任何历史链路时）也一样成立：系统的价值不是"回看历史"，而是"这次判断不再散掉"。从第一条链起，这次判断就被正式承接、可以被下次召回、可以复盘。这个价值从第一天就存在，不需要等历史积累才成立。

## 什么叫高意图

高意图的标志不是用词正式，而是输入背后有真实资金在等待判断。

"ETH 现在还拿吗"是高意图；"ETH 最近走势怎么看"不一定是。边界模糊时，宁可误判为高意图，也不要漏掉。

## 核心对象

`判断链` 是这个系统的核心对象。任何时刻，一条链至少能回答：

- 现在在看什么
- 当前判断是什么
- 判断建立在什么依据上
- 什么变化会触发下一次更新或结束

一次高意图问题被正式承接时，链开始形成；真实管理需求结束后，链才结束。结束后可以进入复盘，但复盘不改变"结束"的定义。

系统真正积累的是两层东西：

- `决策资产`：历史判断、执行结果、复盘结论，以及已确认的偏好边界——默认保留，优先保证链路不丢
- `方法层`：会直接影响后续默认行为的稳定规则——默认保守升级，优先避免把偶然经验过早写成长期规则

两者的分界不是黑盒：明显带有情境性的经验留在决策资产；足够稳定、值得影响后续默认行为的才进入方法层。升级权必须慎重、可见、可回退。

## 核心立场

- 只要问题明显指向真实资金决策，系统就先承接
- 系统先承担大部分上下文重建成本，不把整理负担压回给用户
- 只有真正影响判断质量的关键歧义，才值得再抛回确认

回拉不是可有可无的提醒，而是正式链路的一部分。只要链仍处于活跃管理状态，系统就应该对"下一次为什么值得回来"有默认承接。

## 最难的问题

最难的不是回答市场，而是判断这次输入是在延续旧链还是开启新链。

这是系统最核心的未解风险。押注不是"一开始就能完美判对"，而是：只要能在大多数高意图场景里先检索出足够小的候选旧链集合，再用一次低摩擦确认完成挂接，就已经足够可用。

如果长期做不好，最小可接受降级是：系统仍然负责重建候选上下文，但把链路归属确认明确交还给用户。如果连这种半自动承接都不成立，这个方向就不再是"判断链工作台"，只剩下一次性分析工具。

## 产品成立依赖的关键假设

以下假设如果不成立，方向需要重新评估：

1. **行为改变可以发生**：用户在真实高压决策时刻仍会先进入这个工作台，而不是直接操作。如果这个习惯建立不起来，入口不成立。
2. **检索质量足够可用**：新链/旧链归属依赖检索，如果候选结果长期不准、每次都需要大量纠正，"低摩擦"就是假的。
3. **早期价值足以支撑坚持**：系统价值随历史沉淀增长，但如果冷启动阶段价值不足，用户会在积累起来之前放弃。

## 成功长什么样

最终要帮助我在长周期里更稳定地赚钱：

- 是否比过去更少重复犯同一种判断错误
- 是否能更稳定地区分判断问题、执行问题和情绪问题
- 结果是否更稳，而不是只靠几次运气好的收益

当前阶段的可观测 proxy：在一个可比的 30 天窗口里，高意图决策问题中有多少被承接成了可回看的判断链，并至少发生过一次有效更新、执行记录或复盘回看。更关键的是，这种承接是否真的减少了从零重建上次判断的成本——如果链虽然建起来了，但我仍然经常需要重新解释自己，这个方向依然不成立。

## 当前边界

当前主战场是加密交易协作，运行在 agent 工作区里的持久化 skill 集合。

其他市场信息只进入两种层级：

- 作为加密判断的背景输入
- 在明确要求时作为单次分析对象

默认不进入完整判断链、方法层和持续复盘闭环。
