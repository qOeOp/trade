# 用户故事

## 1. 使用方式

- 只记录真实使用过程，用来校验流程；不定义 UI、字段、表结构或长期制度。
- 某个流程节点没有故事，或故事走不通、不可恢复，先补故事或改流程。
- 当前只覆盖高频、高风险场景，不追求一次写尽。

### US-00 重新进入系统时先恢复并对账当前交易链

- 触发：用户重新进入系统，请求继续当前交易，或系统定时恢复当前语境。
- 核心问题：当前真实状态是什么；哪些信息可信；当前链应从哪里继续。
- 流程要求：先对齐持仓、挂单、已知计划、已有判断、资金状态与最新市场语境，再进入 `OBSERVE / PLAN`、`WAIT-UNTIL-FILL / RECOVERY` 或 `MANAGE`；`空仓` 不能直接等于“无事可做”。

## 2. 开仓前

### US-01 明确标的或先扫机会

- 触发：用户带着明确标的进入，或不指定标的、要求先扫描再收敛；也可能先带着研究结论、宏观判断、关键事件解读或外部材料进入，要求先判断它应映射到哪些标的、方向或本轮是否值得做。
- 用户入口用例：
  - 单标的观察：“看看 `btc` 现在还能不能做多。”
  - 全市场扫描：“现在市场里最值得看的机会是什么？”
  - 带研究结论进入：“我看到有人说 `sol` 这一轮会强于大盘，你帮我判断要不要跟。”
  - 带宏观材料进入：“这份美联储纪要偏鹰，会怎么影响 crypto？”
  - 带关键事件进入：“中东局势升级，油价暴涨，这对 `btc` 有什么影响？”
  - 带外部观点进入：“某个博主说这里是抄底区，你帮我判断靠不靠谱。”
- 当前语境：系统此时还不应默认“用户已经决定要做哪一笔”。用户给出的标的、事件、观点、材料或结论，首先只是线索，不等于已经可直接生成计划。进入 `OBSERVE` 后，系统仍要默认补齐当前市场语境，包括多周期行情、关键结构、常用指标、相关市场与已有上下文；若用户带入的是事件、材料、外部观点或研究结论，`OBSERVE` 还需要调用 `RESEARCH` 做一轮求证和映射，再进入计划形成。这里的计划不只包含 `做哪一笔`，也包含 `不做 / 空仓 / 延后 / 只观察` 这类正式计划输出。
- 核心问题：现在哪些机会值得看；若只做一单，做哪一个；当前研究结论、宏观材料或事件语境是否足以改变优先级、方向判断或 `不做 / 空仓` 结论。
- 系统必须支持：
  - `OBSERVE` 要能接住不同用户入口，但阶段内部默认都要拉取基础判断材料：多周期行情、关键结构、常用指标、相关市场语境和当前链上下文
  - 若用户带入的是 `事件 / 材料 / 外部观点 / 研究结论`，`OBSERVE` 不能直接把它们当结论使用，而要调用 `RESEARCH` 做求证，并映射回当前标的、方向、风险和失效条件
  - `OBSERVE` 不能只堆信息，而要把拿到的数据和求证结果整理成后续能用于判断的材料：`支持证据`、`反方证据`、`风险提示`、`失效条件`
  - 若同时出现多个候选机会，`OBSERVE` 要能先并列保留，再比较优先级，而不是过早只保留一个结论
  - `OBSERVE / PLAN` 至少要把结论拆成两层：`这个机会本身是否成立`，以及 `当前是否适合参与`
  - 当进入 `PLAN` 时，至少要形成一个最小轮廓：`做哪个机会或先不做`、`主要依据是什么`、`当前主要风险是什么`、`什么情况下这轮判断作废`
  - `不做 / 延后 / 空仓 / 只观察` 必须是正式输出，而不是默认沉没状态
- 不宜过早收敛为单一具体交易计划的情况：
  - 用户当前主要在询问影响、可信度或语境翻译，尚未明确要求交易建议；此时系统可以先给出一版 `OBSERVE / RESEARCH` 结果，再继续询问是否要收敛成交易计划
  - 用户输入里包含关键事件、材料、外部观点或研究结论，且 `OBSERVE` 尚未完成必要的 `RESEARCH` 求证和映射；在完成补齐前，不应直接把它们落成单一计划
  - 同时存在多个可考虑机会时，系统可以先给出多个候选计划，并说明各自依据、风险、失效条件和优先级；若要进一步只收敛为一笔，再结合用户约束继续筛选
- 正式输出：
  - 进入某个标的或某组机会的后续 `PLAN`
  - 保留多个候选机会，继续在 `OBSERVE / PLAN` 中收敛
  - 形成 `不做 / 延后 / 只观察 / 空仓` 结论
  - 把需要继续补充的研究问题回交给 `RESEARCH`
- 回退 / 恢复路径：
  - 若只是信息还不够，留在 `OBSERVE`
  - 若出现上下文断裂、已有计划与现实状态冲突，先进入 `RECOVERY`

### US-02 用户不完全接受系统建议但仍想继续

- 触发：系统给出计划，但明确提示条件、盈亏比、胜率、位置或执行方式不理想；用户仍想继续，或认可方向但想改执行方式。
- 核心问题：用户知情后是否仍继续；当前分歧是方向分歧、风险分歧，还是执行方式分歧。
- 流程要求：区分 `风险提示` 和 `硬性阻断`；记录系统原建议、用户确认和最终执行方式；复盘时能拆开方向问题与执行问题。

### US-04 挂单或改单未决时要不要继续等

- 触发：开仓请求已发出但是否真的进市场未定，订单未成交或快成交，撤单 / 改单在途，旧单与新单谁生效未定，或订单因时效、场所约束、执行规则先失效。
- 核心问题：当前到底有没有活跃订单、已成交量和剩余暴露；原机会仍有效，还是只是原执行方式失效；当前挂单所对应的完整交易计划是否仍成立，包括入场、止损、止盈、分批、移动规则和失效条件；现在该继续等、撤单、改价、重挂，还是回到重做计划。
- 流程要求：`WAIT-UNTIL-FILL / RECOVERY` 需要支持 `提交结果未决`、`撤单 / 改单在途`、`订单已失效但 thesis 未失效` 等正式状态；在订单回报、活跃挂单和已成交数量未对齐前，不能把它直接当成失败或完成；若继续保留或重挂，必须说明当前沿用的是原完整计划、还是只沿用方向判断但已改动入场、止损止盈、分批或移动规则，并记录改动理由；支持 `继续等确认 / 先按最坏暴露处理 / 撤回重复单 / 按已成交部分重算后再挂 / 改执行方式后再挂 / 正式删除并回到 OBSERVE / PLAN`。

### US-13 拿到计划后要求展开依据与执行区间

- 触发：用户拿到计划后，要求展开方向、位置和执行方式的依据，并把多周期结构、结构区、执行区间、趋势线、斐波那契位、指标、最强反例、失效条件和明确 `不做` 条件放进同一语境。
- 核心问题：哪些证据和信号足以支撑当前完整计划；哪些只是参考；入场、止损、止盈、分批和移动规则分别由什么理由支撑；哪些冲突足以转成 `不做 / 延后` 或重做。
- 流程要求：`OBSERVE / PLAN` 输出必须可解释、可追溯，并并列呈现支持证据、反方证据、失效条件与 `不做 / 延后` 分支；交易计划不能只输出方向和区间，还要把入场、止损、止盈、分批、移动规则与各自依据放在同一语境里；需要把多周期结构和信号映射回可执行区间，并区分参考项与执行项。

### US-23 多个机会并存时先确认总暴露和真实火力

- 触发：同时存在多个候选机会，或看似不同的机会本质上押同一类风险；已有底仓、对冲仓、战术仓、跨账户仓位、挂单和保证金占用，且资金还可能在划转、赎回、借入或释放途中。
- 核心问题：真实可用火力有多少；当前已有哪些净暴露和相关暴露；应优先做哪一个，哪些不该并存，哪些要等资金或旧暴露释放后再做。
- 流程要求：`OBSERVE / PLAN` 需要支持候选机会排序、相关性和共同因子判断，并汇总跨账户净暴露、挂单占用和可用保证金；区分 `理论能做` 与 `资金已结算可执行`，支持 `保留代表性最好的一笔 / 等到账再做 / 缩规模改做 / 只处理战术仓 / 取消计划`。

### US-24 多腿或对冲式开仓时要按整体计划判断

- 触发：用户计划做 `pair`、`basis`、对冲式入场、跨合约 / 跨场所 / 跨标的多腿同时开仓，或原本看似单腿的机会只有在另一腿也成立时才真正成立。
- 核心问题：这是不是一个必须按组合而不是按单腿理解的机会；每条腿的角色、进入顺序、对冲比和最坏风险是什么；若只成交一部分，当前算已建立组合、失配暴露，还是执行失败。
- 流程要求：`OBSERVE / PLAN` 需要把多腿机会表达成一个整体交易计划，而不是几笔彼此独立的单；明确组合 thesis、各腿角色、允许的先后顺序、每条腿的入场与保护、部分成交后的处理、失效条件与退出条件；`WAIT-UNTIL-FILL / RECOVERY` 需要支持 `某腿已成交 / 某腿未成交 / 某腿失效` 的正式状态，并支持 `补齐另一腿 / 撤回已成交腿 / 临时换载体补对冲 / 整体放弃并回到 OBSERVE / PLAN`。

### US-29 原计划反复回来复查时是否仍有效

- 触发：原计划要求等条件满足后再建仓，后来又出现反复确认、瞬时触碰、插针收回、不同场所触发不一致、错过原入场点、结构未破但环境或事件语境已变，或同一标的旧计划未失效时又冒出新计划。
- 核心问题：原计划现在算有效触发、歧义触发、已错过、已被新计划覆盖，还是应正式作废重做。
- 流程要求：区分 `等待条件成熟` 和 `订单等待成交`；支持 `有效触发 / 歧义触发 / 继续等待 / 原机会已错过不追 / 旧计划被新计划替代 / 正式删除并回到 OBSERVE / PLAN`，且环境变化和事件切换也能触发重做。

### US-30 提前做快速行情和离线分支预案

- 触发：用户担心行情太快来不及交互、将离线一段时间，或认可方向但只想先试仓、分批建仓、梯度挂单、确认后再加仓。
- 核心问题：快速拉升、急跌、假突破或关键时间窗出现时应怎样处理；首层仓位放哪里，后续仓位和条件分支怎么接；不同分支下止损、止盈、移动保护和继续加减仓规则是否已提前定义清楚。
- 流程要求：`PLAN` 需要支持条件分支、离线预案和 `初始仓位 + 后续条件仓位` 的整体计划，区分计划内分批建仓与后续临时追加；离线预案不能只写入场层，还要把对应分支下的止损、止盈、移动规则和切换理由一起带上。

### US-43 机会成立但不一定值得现在做

- 触发：方向看起来成立，但流动性、滑点、历史样本、交易载体或执行成本不理想，或用户连续亏损、疲劳、注意力被打断，当天风险预算与可用保证金也接近上限。
- 核心问题：这笔机会是否真的可执行、可承担、值得现在由自己来做；如果做，应换什么载体、仓位和节奏；如果不做，是降级为观察还是直接放弃。
- 流程要求：`OBSERVE / PLAN` 需要把 `机会是否成立` 与 `当前是否适合参与` 拆开；支持 `缩仓 / 分批 / 换载体 / 只观察 / 跳过 / 不参与`。

## 3. 持仓中

### US-05 持仓语境变化后还拿不拿

- 触发：仓位出现较大回撤、短周期快速逆向、大小周期信号冲突，或消息面、外部观点、研究结论、关键事件、时间拖延、持有成本变化持续打断原判断。
- 核心问题：当前该继续持有、减仓、补保护、临时对冲，还是承认 thesis、时效或经济性已经变了。
- 流程要求：`MANAGE` 需要把短期噪音、thesis 变化、事件风险、时间效率和持有成本拆开；说明新信息为何采纳或剔除，以及它如何改变管理动作；允许在未触发硬止损前正式降风险或离场。

### US-06 对冲或成对暴露失配后怎么接

- 触发：原仓位回撤放大，用户考虑反向短单、锁仓，已有对冲腿失配、失效，原本要成对完成的两腿交易只成交了一边，或某一腿盈利后想先拆掉。
- 核心问题：当前真实净暴露、保证金缓冲和最坏风险是什么；现在该继续维持对冲、补齐另一腿、拆回单边，还是直接退出。
- 流程要求：`MANAGE / RECOVERY` 需要按现实已成交部分重算总暴露、净暴露和最坏情况，不能把“原本打算成对”直接当成已经对冲完成；支持 `保留对冲 / 补齐另一腿 / 先一起缩 / 临时换载体补对冲 / 拆回已成交腿 / 整体退出`。

### US-07 计划内加减仓和保护推进时要不要继续按原节奏走

- 触发：仓位已进入管理阶段，价格和时间窗正在沿原计划推进；计划中的分批减仓、继续持有、条件加仓、移动止损、移动止盈、时间止盈或阶段性收口开始逐步触发。
- 核心问题：当前动作是在按原管理计划推进，还是已经变成新的临时判断；完成这一步后，剩余仓位、保护、目标、thesis 和下一检查点分别是什么。
- 流程要求：`MANAGE / OBS-POS / PLAN-POS` 需要支持把计划内加减仓、移动保护、局部兑现后继续持有和阶段性闭合作为正式路径；每推进一步，都要重算剩余仓位、当前有效保护、剩余风险、后续条件分支和下一检查节奏，并区分 `按原计划推进` 与 `正式改计划后推进`。

### US-08 持仓中的管理动作已发出但未决时先按现实仓位和保护处理

- 触发：持仓中的加仓、减仓、止损、止盈、调杠杆或其他管理动作已经发出，但订单未成交、撤单 / 改单在途、部分成交后旧保护和新保护谁生效未定，或挂出的退出动作先因规则、时效或场所约束失效。
- 核心问题：当前真实仓位、活跃挂单、真正生效的保护和最坏暴露是什么；原管理计划仍成立，还是只是原执行方式失效；现在该继续等、撤单、重挂、补保护，还是回到重做管理计划。
- 流程要求：持仓中的 `WAIT-UNTIL-FILL / RECOVERY` 需要显式支持管理动作和退出动作的未决状态；在订单回报、活跃挂单、剩余仓位和保护链未对齐前，不能把“已经发出减仓 / 止损 / 止盈 / 调整保护”直接当成已经生效；支持 `继续等确认 / 先按最坏暴露处理 / 撤回重复退出单 / 按已成交部分重算仓位与保护后再挂 / 改执行方式后再挂 / 回到 OBS-POS / PLAN-POS`。

### US-10 用户不完全接受系统管理建议但仍想继续

- 触发：系统建议减仓、退出、不要再加仓、提前保护利润、上移止损、取消后续加仓或暂停继续管理，但用户仍想继续持有、继续加仓、推迟止损、改变止盈方式，或只接受方向判断、不接受当前管理节奏。
- 核心问题：当前分歧是 thesis 分歧、风险分歧、节奏分歧，还是执行方式分歧；用户知情后到底在承担什么新增风险；这次偏离是在延续原计划，还是已经构成新的管理偏离。
- 流程要求：区分 `风险提示` 和 `硬性阻断`；记录系统原管理建议、用户确认和最终管理动作；凡是用户拒绝系统的降风险或保护建议，都要重算当前最坏暴露、真实保护和剩余计划，并让复盘能拆开方向问题、风险问题与执行偏离。

### US-09 定期体检或中断恢复时先对账再排优先级

- 触发：用户定期回来看整体持仓健康度，要求系统按时间表拍快照后接续管理，离线回来后状态已变化，或同时存在活跃仓位、未完成挂单、候选机会和研究线程；也可能怀疑保存的数据已有重复、缺失、冲突或过期残留。
- 核心问题：刚才发生了什么；哪些状态现在可信；第一优先级该处理什么。
- 流程要求：恢复路径需要支持快照、事件回放、状态对账、可信度判断、数据健康检查和按风险时效排序；在关键仓位、挂单、保护和管理动作未对齐前，不应直接推进后续管理；至少区分 `可直接修正 / 需人工确认 / 应归档清理 / 暂时冻结不用`，以及 `先处理活跃风险 / 再处理待执行订单与条件 / 最后回到观察与投研`。

### US-31 现实仓位、剩余挂单和保护链对不上时先接管

- 触发：订单部分成交后还留有剩余挂单，父单开始成交但附带 `TP/SL`、bracket 或其他保护未必已生效，用户手动加减仓、改保护、补保证金、切账户模式后再让系统接管，仓位数量变化后保护链可能失配，或仓位已经没了但旧退出单仍可能活着。
- 核心问题：当前真实仓位是什么；哪些挂单和保护还有效；哪些暴露未被保护，哪些退出单已经变成孤儿单。
- 流程要求：继续 `MANAGE` 前要先对齐现实仓位、剩余挂单、当前真正生效的保护链和计划偏离；不能把“下单时附带过保护”或“之前设过保护”直接当成当前全部受保护；支持 `保护仍匹配 / 部分失配 / 先补保护 / 先撤孤儿单 / 直接退出 / 回到新的 OBS-POS / PLAN-POS`。

### US-35 原方向失效后要不要直接反手

- 触发：用户认为原方向逻辑被打破，同时出现了反方向机会。
- 核心问题：是先结束旧仓位再重新观察，还是可以直接从多变空、从空变多。
- 流程要求：系统要把 `结束旧仓位` 和 `开启新方向` 拆成两个判断，避免把反手当成单一动作。

### US-37 降风险与补救动作未确认时按当前最坏暴露处理

- 触发：仓位接近爆仓、可用保证金不足、补钱或调仓资金在途，想减仓、补保护、切换退出方式却先被已有占用、规则约束或账户模式挡住，交易所执行异常、场所规则变化，或已经发生部分强平、自动减仓、保险基金接管后只剩残余状态。
- 核心问题：当前真实暴露和最坏风险是什么；哪些补救动作只是已经发起，哪些已经真正生效；现在该先保命、迁移、对冲，还是直接退出。
- 流程要求：`MANAGE / RECOVERY` 需要把 `补救意图已发出` 和 `补救已生效` 拆开；在途资金、在途改单或异常回报都不能直接当成保护已到位；先确认真实成交、活跃订单、保护链和场所规则，再决定 `到账前先降风险 / 撤旧单释放占用后重提 / 临时对冲 / 迁移场所 / 直接退出 / 重建剩余仓位与保护`。

### US-46 多次加减仓后这笔仓位还是不是原来那笔

- 触发：已有仓位时又出现第二次入场、新机会或资金轮动需求，用户已部分止盈后想继续拿、回补或重排保护，浮亏来到原设二次入场区，或同一标的经历多次加减仓、对冲、换周期后，用户想把它改口成另一笔交易。
- 核心问题：当前加仓、回补或轮动是计划内延续、合法切换，还是在为错误续命；当前剩余暴露还是不是原来那笔。
- 流程要求：`MANAGE / REVIEW` 需要支持判断当前剩余暴露和当前计划是不是还算原来那笔；凡是加仓、减仓、回补、再对冲或换周期，都要一起重算 thesis、失效条件、总风险、目标和检查节奏，并区分 `计划内分批建仓`、`已兑现部分收益后的剩余仓位` 和 `情绪化摊平`。

## 4. 投研与回测

### US-39 从历史里找类似场景并做回放验证

- 触发：用户想知道当前场景，或某个特定历史时间窗口，以前是否出现过相似案例、事件或走势；也可能要求从某个历史日期开始回放，并一步步模拟当时的完整交易过程。
- 核心问题：过去有哪些类似样本；当时真正重要的事件和处理方式是什么；如果只能看到当时已发生的信息，这套判断还能不能站得住。
- 流程要求：支持把当前问题映射到历史案例和时间锚点，并把相关事件回收到同一语境；`回放` 需要支持 `选起点 / 锁住未来信息 / 生成当时语境下的 OBSERVE / PLAN / 推进时间后继续 MANAGE / REVIEW` 的循环。

### US-56 从历史样本里提炼并验证候选策略

- 触发：用户已经积累了一批真实交易，也包含没成交、没执行、主动放弃的样本，怀疑其中有重复模式；或带来外部方法、粗糙假设、截图标注、过滤条件、执行约束与相邻版本，想放到更长历史里做系统检验。
- 核心问题：哪些部分能被翻译成可量化、可重复检验的信号、过滤和执行规则；哪个条件真的带来边际价值；这套想法适用于哪些环境，又会在什么前提下失效。
- 流程要求：`RESEARCH -> STRATEGY -> BACKTEST` 需要支持从历史样本和外部方法中抽出候选规则，对相邻版本做条件隔离，把想法拆成可量化信号、过滤条件、调仓规则、成本假设和评价指标，再检查时点、滑点、流动性和成交假设是否破坏真实可执行性；输出适用边界、失效环境，以及它应进入主线、转成观察清单或影子跟踪、继续留在 research，还是只适合回放验证。

### US-73 消化外部材料并把研究结论回接主线

- 触发：用户要求系统解读宏观材料、市场报告、跨资产关系、某标的相对同类资产的异常强弱，或持续记录多市场线索后再做周期分析；也可能要求系统主动找当前 thesis 的反例和失效证据。
- 核心问题：这些外部信息和长期线索里，哪些是真正重要的新语境、风险提示或研究结论；哪些只是背景噪音；哪些反证足以推翻原判断。
- 流程要求：`RESEARCH` 需要支持 `消化外部材料 / 汇总长期线索 / 主动寻找反证 / 提炼结论 / 映射到具体标的`；输出既可服务 `OBSERVE / PLAN / MANAGE`，也可继续沉淀为候选研究结论或候选策略。

## 5. 解释、复盘与迭代

### US-12 闭链进入待复盘池后什么时候拉出来复盘

- 触发：一条交易链已经完全结束，或虽然没有彻底结束但已经阶段性闭合；用户可能想立刻复盘，也可能只是先留待后面统一回看。
- 核心问题：这条闭链现在是否已经满足复盘条件；应立刻复盘，还是先进入待复盘池；之后要按什么线索把它从池子里再取出来。
- 流程要求：凡是已结束或阶段性闭合的交易链，都需要进入待复盘池，不要求必须即时复盘；`REVIEW` 可以在闭链后立即触发，也可以在后续按时间、主题、异常程度、收益结果或用户指定条件，从池子中选择闭链进入复盘；只要池子非空，系统就应能继续拉取闭链复盘，而不是把“没马上复盘”当成链路断掉。

### US-14 用户追问这笔交易为什么这样建议、后来为什么改

- 触发：用户要求系统解释某笔交易当时为什么给出那个建议、为什么后来改了计划、为什么提前或延后执行、为什么止损止盈或保护方式发生变化，或直接追问到底是哪条证据、哪次事件、哪段判断改变了动作。
- 核心问题：系统当时的原判断是什么；后来的新判断是什么；两者之间究竟是哪条证据、哪段语境或哪项约束发生变化，才导致建议、计划或动作改变。
- 流程要求：`解释 / REVIEW` 需要支持沿着同一条交易链回看 `当时看到什么 / 当时怎么判断 / 后来新增了什么 / 为什么采纳或放弃 / 因而改了什么动作`；解释不能只给结论，必须能把计划变化映射回具体证据、反证、事件、约束和用户确认；至少能区分 `原判断未变只是执行方式变了 / thesis 变了 / 风险约束变了 / 交易场所或成本约束变了 / 用户主动偏离系统建议`。

### US-15 迭代后的策略版本何时切回主线使用

- 触发：一条策略经过 `REVIEW -> BACKTEST -> ITERATE` 后产生了新版本，或原版本被收紧、放宽、合并、降级观察、暂停或淘汰；用户想知道下一轮观察和计划到底应沿用哪个版本。
- 核心问题：这次迭代产物是否已经足以替代旧版本进入主线；当前主线应读取哪个生效策略版本；旧版本是继续保留作对照、降级备用，还是退出后续主线使用。
- 流程要求：`ITERATE` 需要把复盘和回测结论收敛成带版本的策略更新，并让后续 `OBSERVE / PLAN` 明确使用当前生效版本，而不是混用新旧规则；同一策略可以保留多个版本供比较和回看，但主线在任一时点只能有清晰的当前版本；至少支持区分 `升级为当前版本 / 保留旧版对照 / 降级为观察清单 / 暂停使用 / 淘汰`。

### US-11 阶段性复盘时先判断交易和策略是否真的在变好

- 触发：用户定期回顾一段时间内的交易、事件、决策和策略调整，或在单笔大赚大亏、连续盈利后，开始怀疑结果是否过度依赖少数样本、运气、违规盈利或同一类集中风险；也可能感觉候选策略已经太多、边界模糊。
- 核心问题：系统到底有没有变好；该保留的是哪类 edge，还是只是在堆叠样本、放大运气和策略冗余。
- 流程要求：`REVIEW / ITERATE` 需要把收益、样本分布、过程质量、组合集中度和策略池重叠度拆开；支持 `保留 / 合并 / 降级观察 / 暂停 / 淘汰`，且任何放大仓位或放宽过滤条件的决定都要绑定可解释证据。

### US-20 止损后或同一 thesis 多次重试后，要不要再上

- 触发：刚止损后行情很快又回到原方向，或同一标的、同一结构、同一核心 thesis 在短时间里已经止损、被洗、错过后追回，用户仍想继续等下一次同类机会。
- 核心问题：这次重入是基于新证据的纪律性机会，还是已经滑向报复性重试、过度关注单一想法。
- 流程要求：允许 `REVIEW` 与下一轮 `OBSERVE` 快速衔接，但必须把 `基于新证据的重入` 和 `情绪化追回` 拆开；多次重试要被视为同一想法的连续样本，而不是几笔彼此无关的交易。

### US-64 连续亏损或事故后，是继续、缩手还是暂停

- 触发：用户连续止损、短期回撤扩大、执行动作变形，暂停一段时间后准备恢复实盘，或刚经历强平、自动减仓、穿仓等事故级亏损。
- 核心问题：现在是继续、降风险继续、继续观察，还是先暂停；若要恢复，先看到哪些证据才算恢复条件成立。
- 流程要求：`REVIEW / RECOVERY / ITERATE` 需要把 `暂停决定 / 事故链复盘 / 恢复门槛` 连成一条路，并支持 `继续观察 / 影子恢复 / 小仓恢复 / 恢复常规` 的分级路径。

### US-129 方向看对却没赚到，edge 到底丢在哪个环节

- 触发：一段时间里，用户反复在事后看对方向或结构，但真实结果表现为没成交、没按计划执行、拿不住、过早止盈、追着回补，或回测看起来不错而实盘被成本、成交和管理链路吃掉。
- 核心问题：edge 真丢在观察、计划、下单、成交、持仓管理、恢复衔接还是情绪执行；最该修的是 thesis、执行链路、交易载体、持有经济性还是风险预算。
- 流程要求：`REVIEW / ITERATE` 需要把 `系统原计划 / 用户实际执行 / 最终结果` 持续对照，并把“看对了”和“赚到了”之间的链条正式拆开，至少区分 `没等到 / 不敢做 / 做了但仓位不对 / 拿不住 / 保护失真 / 恢复断档 / 成本吃掉 edge` 等节点。
